The website

The ASL website is accessible at asl.stanford.edu . It is hosted on Stanford infrastructure. The website runs Wordpress.

The website is on the lab AFS (/afs/ir/group/asl/cgi-bin/wordpress/). To gain access to the AFS, a user that is currently a member of asl-admins should log into any FarmShare system (e.g., cardinal.stanford.edu) and run the following command for each new admin, replacing SUNETID as appropriate:
pts adduser -user SUNETID -group asl-admins
To gain access to the Wordpress install, a current admin should add the user from the WP panel.

The current sysadmins for the ASL website are Ed Schmerling (schmrlng) and Federico Rossi (frossi2).

Website load times are sometimes slow. We believe this is due to the host machine. This page monitors load times every 15 minutes since August 2015.

Segovia

Segovia is a HP Proliant DL380e Gen8 server running Windows Server 2012. It is located in Rack 3/11-12U of the Huang 002 datacenter. Access to the datacenter and hardware support is provided by School of Engineering IT (SoEIT).

Segovia is accessible via Microsoft Remote Desktop at segovia.stanford.edu . Access via Remote Desktop is only allowed from the Stanford VPN. To get an administrative account on Segovia, talk to the current sysadmins.

Segovia hosts a SVN server (served through https at https://segovia.stanford.edu/svn/) managed with VisualSVN. The SVN server is accessible from the whole Internet. Segovia also hosts a FTP-SSL server (served through ftpes://segovia.stanford.edu) managed with FileZilla Server. The FTP server is accessible from the Stanford VPN only.

The iLOM module for Segovia, which allows to shut down and restart the machine remotely, is accessible from the Stanford VPN at segovia-ilom.stanford.edu. Prof. Pavone is the custodian of the login credentials.

The current sysadmins of Segovia are Federico Rossi (frossi2) and Andrew Bylard (bylard). Note: all sysadmins should be subscribed to the hecserverroomoccupants mailing list, managed by SoEIT.

Phobos

Phobos is a custom-build desktop machine running Ubuntu 16.04. It is hosted in Durand 009 (under Stefan's desk).

Phobos is accessible via ssh at phobos.stanford.edu . Access is only allowed from the Stanford VPN. Note that you need to be on the Stanford VPN even if you are connecting from campus! To get an account on Phobos, talk to the current sysadmins.

Phobos does not currently expose publicly available services other than ssh. Individual users may have their own services exposed to the Stanford VPN. Phobos also hosts a full backup of Segovia's ASL SVN repository (at the time of writing, only the ASL repository is backed up).

Phobos does not have a iLOM module.

The current sysadmins of Phobos are Ed Schmerling (schmrlng) and Federico Rossi (frossi2).

GPU workstations: Io, Europa, Ganymede, and Callisto

The lab has 4+1 workstations for GPU-heavy workloads. Io, Europa, Ganymede and Callisto have AMD Ryzen 1700X processors and a combination of NVIDIA GTX 1080 and Titan graphics cards. Ceres is a small-form-factor computer for embedded automotive applications.

The sysadmins are James (Io), Ed (Europa, Callisto, and Ceres), Brian (Ganymede), and Karen (Callisto and Ceres). Access policies to the GPU workstations vary: to get access to one or more of the machines, ask the sysadmins.

Other machines: Deimos, Skynet and Sputnik

Deimos, Skynet and Sputnik are Windows workstations hosted in Durand 009b (Deimos and Skynet) and Durand 010 (Sputnik).

Skynet acts as the server for the Vicon system installed in 009b. Sputnik acts as the server for the Optitrack system installed in Durand 010. Deimos is primarily a CAD machine.

Deimos, Skynet an Sputnik are not remotely accessible and they have a single user, "ASL". There is no designated sysadmin for these machines. To gain (local) access to one of these machines, talk to any lab member.
